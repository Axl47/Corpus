Local-First Billing Removal

Status: implemented, 2026-06-21. Owner: Codex. Outcome: Corpus no longer exposes Stripe, billing, pricing, subscriptions, or plan-management surfaces in active runtime code. Runtime quota checks now allow local-first use with the former Enterprise/unlimited behavior.

Current Risk

Billing logic is used as a quota gate inside invites, uploads, workspace creation, and agent-token/OAuth flows. Removing UI alone would leave hidden limits.

Decision

Delete payment surfaces and make quota checks no-op/unlimited. Keep legacy DB artifacts only when dropping them would require a risky migration from partial Drizzle history.

Validation

npx tsc --noEmit passes. Reference scans are clean. npm run lint remains blocked by existing repo-wide React compiler/script lint failures unrelated to payment removal.

Context

Stripe support currently spans src/lib/stripe.ts, src/lib/billing/*, src/lib/actions/billing.ts, src/app/api/webhooks/stripe/route.ts, the stripe npm dependency, and environment variables. User-facing plan and billing surfaces appear in workspace navigation, mobile account actions, user settings, admin user detail, workspace settings, landing pricing, sitemap, auth public-route allowlists, and translations.

The generated Drizzle migration history only covers through src/db/migrations/0018_user_sessions.sql; later changes, including billing, are manual scripts. For this pass, active schema exports were cleaned up, but no destructive data migration was generated.

Phases

  1. Remove Stripe and billing entrypoints: delete webhook route and payment helpers, uninstall stripe, remove Stripe env examples.
  2. Convert quota gates: replace billing service imports with local-first limits helpers that always return allowed and expose only non-payment people helpers where needed.
  3. Remove UI surfaces: delete billing/pricing components and remove associated buttons, tabs, modals, route links, success query handling, and admin plan controls.
  4. Clean translations and docs: remove unused Billing/Pricing namespaces and plan/billing copy; update AGENTS.md to reflect the local-first model.
  5. Validate with TypeScript, lint, and targeted searches.

Recovery

All edits are file-level and non-destructive to user data. Deleted source files can be restored from git if needed. No database migration is planned in this pass, so existing local data remains intact.
